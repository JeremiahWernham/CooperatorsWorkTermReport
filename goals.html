Goals

  • CRITICAL & CREATIVE THINKING - Inquiry & Analysis

    Create new cmdlets and small GUI-based applications to help automated the day-to-day processes of the team.

    Action Plan

    Inquire about possible team processes that would benefit from small automation tools and analyzing the usefulness of tools I create to hopefully integrate them into everyday work. Before I can begin to develop these tools I need to improve my working knowledge of scripting languages such as Windows Powershell and Python. Taking online certificate courses while the ticket queue in relatively slow would be an effective way of doing this. Using my knowledge acquired in class about user-friendly software development and quickly learning about different libraries and frameworks can help increase the quality and robustness of my scripts. Having a co-worker test and provide feedback about the application would allow me to improve on it as well. This would also allow me to improve existing scripts and applications the team uses.

    Measure of Success

    Contribute to the efficiency of any existing process that the team uses somewhat regularly. I will attempt to create a tool that serves the general workflow or a more specialized application such as for tickets that require me to gather more information about a computer on the network. I can determine if the tool I create is useful by generating survey feedback through an online survey web application and even improve on the application with criticism of the team. Ultimately, I aim to be satisfied with the tool itself given the time and effort to learn and use a new scripting language.

    Reflection

    In my free time, while minimal, Lynda.com offered by the University of Guelph did allow me to learn about Windows PowerShell and Python as scripting languages and expand my knowledge of how existing GUI-based programs that use these languages on my teamwork. With this knowledge I wrote a short program for myself called "Start Up" which would automatically open and order the windows which I needed to start my workday. Thought not having a GUI, I used this program every morning. Secondly, I was able to create short PowerShell scripts that I would use when reporting on different workstations on the Co-operators network. For example, during deployments I would run a script that would return the current version of an application to tell me if they were up-to-date. Overall, I did not find need to create a complicated GUI-based program to improve my team's everyday work, however with what I learned about these languages I was able to create numerous small ones to save a lot of time and energy.

  • COMMUNICATING - Written Communication

    Write or Edit 5 useful and informative knowledge articles per month for users and technical analysts to find solutions for technical issues.

    Action Plan

    Part of my role as a Technical Support Analysts includes documenting new known errors and how to articles with solutions for everyone across Co-operators IT teams to reference and attach to tickets. The articles should include screenshots of the relevant errors and step by step instructions on how to fix them. When a new problem that hasn't been diagnosed arises my objective is to first resolve the problem, sometimes with the discretion of other teammates, and research done on the internet. Next, I type detailed notes on my resolution process and use them to create useful and informative knowledge articles for such errors or inquiries in the future. This requires both a strong understanding of computers and computer software and an observant eye in recognizing the main problem component(s). I aim to have my knowledge articles peer-reviewed and published by my teammates with little or no critical remarks or changes to made.

    Measure of Success

    When a knowledge article is submitted for publication, it is first peer reviewed for proper spelling and grammar, relevance, and quality. My measure of success is to have 90% of my knowledge articles approved on the first submission. I am to author or edit 5 each month though another measure of success is to author several knowledge articles with a view count of more than 20 by the end of the work term. This would ensure that my contributions are readable and relevant for my team members.

    Reflection

    Throughout my term, I was consistent in updating existing knowledge in our database 5 or more times per month. As well, I became proficient in simplifying articles with steps that would resolve the issue more quickly while removing any unnecessary steps toward fixing the problem. I pioneered multiple different problem records for large scale issues that required new documentation to resolve and prevent future incidents. Each pear reviewed and approved with little editing required. Therefore, I was successful in achieving this goal.
